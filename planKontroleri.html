Specifično za nas

Pravila:

  • Arhitektura i određivanje ruta naše aplikacije bazirat će se na načelu Representational state transfer.
  • Jedno od dobrih praksa izgradnja je kontrolera na temelju zajedničke odgovornosti pa smo ih tako i podijelili.
  • Pri zahtjevu s metodom koja se nalazi u stupcu HTTP Metoda upućenom na URI iz stupca Ruta (appenda se na podrazumijevanu rutu), u kontroleru kojem tablica pripada zove se metoda koja će obaviti Funkcionalnost. Ta je funkcionalnost implementacija operacije iz stupca REST ruta, a dio je CRUD funkcije obilježene bojom.
  • ":id" je placeholder za jedinstveni id pojedinog entiteta.

SessionController

Sve rute podrazumijevaju putanju/sessions
ime REST rute Funkcionalnost Ruta HTTP Metoda Povratna vrijednost Ime viewa
New Prikazuje stranicu s formularom za prijavu "/new" GET View login
Create Prijavljuje korisnika, započinje sjednicu "/" POST Redirect profile
Destroy Odjavljuje korisnika, briše sjednicu "/" DELETE Redirect index

UserController

Sve rute podrazumijevaju putanju/users
ime REST rute Funkcionalnost Ruta HTTP Metoda Povratna vrijednost Ime viewa
Index prikazuje listu korisnika "/" GET View users
New prikazuje formular za registraciju "/new" GET View register
Create dodaje novog korisnika "/" POST Redirect profile
Show prikazuje detalje korisnika "/:id" GET View profile
Edit prikazuje formular za uređivanje "/:id/edit" GET View editUser
Update sprema promjene nastale uređivanjem "/:id" PUT Redirect profile
Destroy briše korisnika "/:id" DELETE Redirect admin -> users, drugi -> index
