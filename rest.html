Specifično za nas

Pravila:

  • Arhitektura i određivanje ruta naše aplikacije bazirat će se na načelu Representational state transfer.
  • Jedno od dobrih praksa izgradnja je kontrolera na temelju zajedničke odgovornosti pa smo ih tako i podijelili.
  • Pri zahtjevu s metodom koja se nalazi u stupcu HTTP Metoda upućenom na URI iz stupca Ruta (appenda se na podrazumijevanu rutu), u kontroleru kojem tablica pripada zove se metoda koja će obaviti Funkcionalnost. Ta je funkcionalnost implementacija operacije iz stupca REST ruta, a dio je CRUD funkcije obilježene bojom.
  • ":id" je placeholder za jedinstveni id pojedinog entiteta.

SessionController

Sve rute podrazumijevaju putanju /sessions
ime REST rute Funkcionalnost Ruta HTTP Metoda
New Prikazuje stranicu s formularom za prijavu "/new" GET
Create Prijavljuje korisnika, započinje sjednicu "/" POST
Destroy Odjavljuje korisnika, briše sjednicu "/" DELETE

UserController

Sve rute podrazumijevaju putanju /users
ime REST rute Funkcionalnost Ruta HTTP Metoda
Index prikazuje listu korisnika "/" GET
New prikazuje formular za registraciju "/register" GET
Create dodaje novog korisnika "/" POST
Show prikazuje detalje korisnika "/:id" GET
Edit prikazuje formular za uređivanje "/:id/edit" GET
Update sprema promjene nastale uređivanjem "/:id" PUT
Destroy briše korisnika "/:id" DELETE

PetController

Sve rute podrzumjevaju putanju users/:id/pets
ime REST rute Funkcionalnost Ruta HTTP Metoda
Show prikazuje popis ljubimaca u sklopu userController-a -
New prikazuje formular za dodavanje ljubimca "/new" GET
Create dodaje novog ljubimca "/" POST
Destroy briše ljubimca "/:id" DELETE

ReservationController

Sve rute podrazumijevaju putanju users/:id/reservations
ime REST rute Funkcionalnost Ruta HTTP Metoda

New prikazuje formular za rezervaciju "/:id/new" GET

Show prikazuje rezervaciju detaljno "/:id/edit" GET
Edit prikazuje formular za uređivanje "/:id/edit" GET
Update spremi rezervaciju "/:id" PUT
Destroy otkaži rezervaciju "/:id" DELETE
